# How to write a PR

Your PR must have the following sections:

```markdown
## Overview

(Copy directly from the corresponding planning file)

## Happy Flow

(Copy directly from the corresponding planning file)

## Data Flow

(Copy directly from the corresponding planning file)

## Manual Verification

(Copy directly from the corresponding planning file)

```

For any UI changes, please add the before/ and after/ pictures from the docs/planning/<folder>/ paths, in the following way:

```markdown

## State (Before)

(Screenshots)

## State (After)

(Screenshots)

```

If you don't have these pictures, prompt the user for further instruction.
